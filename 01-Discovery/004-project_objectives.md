# Project Objectives

**Project:** Project SentinelCare - Cyber Security Governance, Risk and Compliance Improvement Programme
**Document Type:** Project Objectives
**Classification:** Portfolio Case Study – Fictional Organisation
**Version:** 1.0

# 1. Purpose

This document sets out the objectives for Project SentinelCare's Discovery phase. It translates the business drivers described in [003-business-context](003-business-context.md) into specific, assessable objectives, each linked to the relevant NCSC Cyber Assessment Framework (CAF) outcome and to the maturity scale defined in [001-executive_brief](001-executive_brief.md) (Section 9).

Objectives are stated qualitatively rather than against fixed dates or numeric targets. Discovery is a point-in-time assessment; committing to delivery dates or hard maturity scores is more appropriate once a baseline has actually been established and a treatment plan agreed with the Trust Board.

# 2. Objectives

| ID | Objective | Baseline (current state) | Target direction |
|---|---|---|---|
| O-01 | Establish a consistent, Board-approved cyber security governance framework | Policies exist but are applied inconsistently (Level 1–2, Initial/Developing) | Move toward Level 3 (Defined): approved, consistently applied policy set with clear ownership |
| O-02 | Establish a central cyber risk register with defined ownership | No central register; risk ownership inconsistent | Level 3: risks identified, scored, and owned by named business/system owners |
| O-03 | Improve visibility and control of information assets, including medical devices | Incomplete asset inventory; legacy devices with unclear ownership | Level 2–3: consolidated asset register covering clinical, corporate, and connected medical device estate |
| O-04 | Strengthen identity and access management | Excessive privileges identified in internal review | Level 3: role-based access, periodic access reviews, privileged access management in place |
| O-05 | Establish a proportionate supplier security assurance process | No central supplier assessment process across 150+ suppliers | Level 2–3: risk-tiered supplier assessment applied to new and existing contracts |
| O-06 | Improve data governance and classification practice | Inconsistent classification; data held outside approved systems | Level 2–3: classification scheme applied to priority information assets |
| O-07 | Test and improve incident response capability | Plan exists but is not regularly tested | Level 3: incident response plan exercised at least annually, with lessons learned captured |
| O-08 | Improve executive-level cyber risk reporting | Limited security metrics reaching the Board | Level 2–3: regular Board/Audit and Risk Committee reporting on cyber risk and compliance status |

# 4. Out of Scope for These Objectives

The following are explicitly not objectives of the Discovery phase:

- Delivery or implementation of remediation activity — Discovery identifies and prioritises; delivery is addressed in a subsequent phase.
- Live technical testing, penetration testing, or vulnerability scanning (see constraint C-007 in [007-assumptions_and_constraints](007-assumptions_and_constraints.md)).
- Formal submission of the NHS Data Security and Protection Toolkit (DSPT) — the assessment aligns to DSPT's evidence structure but does not constitute a submission.
- Certification against ISO/IEC 27001 — the assessment maps current practice to ISO 27001 controls but does not represent a certification audit.
